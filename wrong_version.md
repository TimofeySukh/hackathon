# Раскладка групп («электронные облака») — текущее состояние

Снимок того, как **сейчас** работает раскладка blob-групп. Ветка `nested-rings`
(ответвлена от `predprod`). Часть закоммичена (`47190af`), часть — в рабочем
дереве (незакоммичено): `src/App.tsx`, `src/index.css`.

Файл называется `wrong_version.md`, потому что это экспериментальная версия,
которой ещё крутят детали — не финал.

---

## Что видит пользователь

Каждая **группа** (blob) на доске рисуется как **заполненный круг из
концентрических колец** — визуально как электронные оболочки вокруг ядра:

- **Орбиты** — концентрические залитые кольца-полосы (толщина `70`) с
  **прозрачными зазорами** между ними (фон доски просвечивает). Цвет — по тегу.
- **Пустой центр** (ядро) — за него (и за полосы) таскается **весь кластер**.
- **Основная категория зоны** = самый частый тег в группе. Эти люди заполняют
  кольца как обычный круг.
- **«Чужаки»** (люди с тегом ≠ основной категории) собираются в **пирамидку**
  внизу круга: 1 на внутреннем кольце, 3 на следующем, 5 дальше… (треугольник,
  апекс к центру). Остаток каждого кольца — основная категория.
- **Неприкреплённые контакты** (не входят ни в одну группу, кроме корневого
  `You`) — с **серой пунктирной обводкой**.

### Жесты / поведение

- **Перетаскивание за центр/полосы** — двигает всю группу.
- **Перетаскивание точки** — двигает одного человека (на отпускании может
  присоединиться к группе/образовать новую — старая логика `finishNodeGroupDrop`).
- **Дабл-тап по человеку** — **отцепляет его от группы на месте** (никаких
  телепортов). Он получает серую обводку. Оставшиеся участники **переколлекуются
  вокруг прежнего центра**, чтобы зона **не прыгала**. Если в группе осталось
  <2 — она распускается.
- **Смена цвета тега** в палитре → если это основной тег зоны, **зона и её
  орбиты сразу перекрашиваются** (цвет берётся из текущего цвета тега, а не из
  сохранённого снимка).

## Как это устроено в коде (`src/App.tsx`)

### Константы (раздел рядом со строкой ~235–246)
```
GROUP_MEMBER_SPACING   = 72   // расстановка для группы из 2 человек
GROUP_ZONE_WIDTH       = 130  // радиальный зазор между кольцами
GROUP_CORE_RADIUS      = 70   // пустое ядро-ручка в центре
GROUP_ROW_SPACING      = 132  // шаг между людьми вдоль кольца
GROUP_ORBIT_BAND_WIDTH = 70   // толщина залитой полосы-орбиты (< zone → есть зазор)
```
Радиус кольца k: `getGroupRingRadius(k) = CORE + (k + 0.5) * ZONE`.
Ёмкость кольца ≈ `2π·radius / GROUP_ROW_SPACING`.

### Ключевые функции
- **`getGroupPrimaryTagKey(members)`** — основной тег группы (самый частый;
  untagged = ключ `''`).
- **`arrangeGroupMembers(nodes, memberIds, centerOverride?)`** — раскладка точек.
  Делит на `mainMembers` (основной тег) и `outsiderMembers` (остальные,
  отсортированы по тегу), вызывает `placeCircleWithPyramid`. Опциональный
  `centerOverride` — якорь центра (используется при отцеплении, чтобы зона не
  уезжала).
- **`placeCircleWithPyramid(main, outsiders, cx, cy, positions)`** — заполняет
  концентрические кольца: чужаки идут плотным треугольником вниз
  (`pyramidDirection = Math.PI/2`), основная категория — по остатку каждого кольца.
- **`getNodeGroupBubbles(groups, nodes, tagColorById?)`** — данные для рендера
  зоны: `path` (выпуклая оболочка → сглаженный путь), `center` (среднее позиций
  участников), `ringRadii` (радиусы занятых колец), `fill`/`stroke`. **Цвет
  берётся из `tagColorById[primaryKey]`** (если есть), иначе из `group.color`.
- **`detachNodesFromGroups(nodeIds)`** — убирает узлы из memberIds всех групп,
  `sanitizeNodeGroups` + сохранение (remote/local).
- **`handleNodeDetach(node)`** — обработчик дабл-тапа: запоминает центр группы,
  отцепляет, переколлекует оставшихся вокруг этого центра.

### Рендер
- Группы — в SVG `graph-connections` (строка ~5120):
  - невидимый `<path class="node-group-bubble">` (fill/stroke transparent) —
    зона перетаскивания;
  - набор `<circle class="node-group-orbit">` по `ringRadii` — залитые полосы
    (толщина через inline `strokeWidth: GROUP_ORBIT_BAND_WIDTH`, цвет = `fill`).
- Узлы — HTML `GraphNodeCard` поверх SVG. Пропсы `isSelected`, **`isUnattached`**
  (`!is_root && !groupedNodeIds.has(id)`), `onDoubleClick={handleNodeDetach}`.
- `groupedNodeIds` — useMemo: множество всех memberIds.

### CSS (`src/index.css`)
- `.node-group-orbit` — `fill: none; pointer-events: stroke;` (толщина/цвет —
  inline).
- `.graph-node.is-unattached .graph-node__button::before` — серая пунктирная
  обводка (есть тёмная и `.theme-light` версии). `.is-selected` идёт после и
  перекрывает её сплошной рамкой.

### Авто-перекладка
- Одноразовый `useEffect` (рядом с `draggedPositionsRef`), гейт по localStorage
  `hackathon-group-repack = 'group-layout-v8'`. Прогоняет `arrangeGroupMembers`
  по всем группам и пишет позиции **батчами по 6** (чтобы не залить бэкенд).
  **Чтобы заставить перелечь заново — поднять версию (`v8` → `v9`).**

## Бэкенд (важно)

Фича групп требует таблицу **`public.node_groups`** в Supabase
(миграция `supabase/migrations/20260610120000_add_node_groups.sql`). На рабочей
базе (`lxnrpdeahoglgiocowsh`) она была **накатана вручную** через SQL Editor
(минимальная версия без FK/триггеров), т.к. CLI не залинкован.

Код терпит отсутствие таблицы, чтобы доска грузилась/работала и без неё:
- `src/lib/graphStorage.ts` — `isMissingTableError()`; в `loadBoardGraph` и
  `saveNodeGroups` ошибка «нет таблицы» не валит загрузку/запись (группы —
  только в локальном состоянии на сессию).
- `src/lib/useBoardGraph.ts` — ошибки загрузки/записи показывают **реальный
  текст** (раньше маскировались под «Unable to load board data.» / «Board
  update failed.») + лог в консоль.

## Текущие шероховатости / на докрутку

- Значения зазоров/плотности (`GROUP_ZONE_WIDTH=130`, `GROUP_ROW_SPACING=132`,
  `GROUP_ORBIT_BAND_WIDTH=70`) подбирались на глаз — можно крутить.
- Группы на доске у пользователя по факту почти **одно-теговые** (пирамида
  чужаков всплывает редко).
- Дабл-тап также вызывает обычный `onClick` (открывает инспектор) — побочный
  эффект не убран.
- Подписи (имена) при плотной раскладке всё ещё могут налезать.
